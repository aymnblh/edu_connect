# EduConnect — Flutter School Communication App

A production-ready Flutter mobile application connecting teachers and parents with real-time features built on Firebase + Riverpod.

---

## 🚀 Quick Start

### 1. Prerequisites

- Flutter SDK ≥ 3.3.0
- A Firebase project

### 2. Firebase Setup (Required)

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Create a new project (e.g. `edu-connect`)
3. Enable these services:
   - **Authentication** → Sign-in method → Email/Password ✅
   - **Firestore Database** → Start in **test mode** initially
   - **Cloud Messaging** (automatically enabled)

4. Add your apps:
   - **Android**: Register with package name `com.educonnect.edu_connect`
     - Download `google-services.json` → place in `android/app/`
   - **iOS**: Register with bundle ID `com.educonnect.eduConnect`
     - Download `GoogleService-Info.plist` → place in `ios/Runner/`

5. Install FlutterFire CLI and configure:
   ```bash
   dart pub global activate flutterfire_cli
   flutterfire configure
   ```
   This will overwrite `lib/firebase_options.dart` with your real credentials.

### 3. Install Fonts (Inter)

Download from [Google Fonts](https://fonts.google.com/specimen/Inter) and place:
```
assets/fonts/Inter-Regular.ttf
assets/fonts/Inter-Medium.ttf
assets/fonts/Inter-SemiBold.ttf
assets/fonts/Inter-Bold.ttf
```
Or temporarily remove the font config from `pubspec.yaml` to use system fonts.

### 4. Run the App

```bash
flutter pub get
flutter run
```

## 🔥 Firestore Structure

```
users/{userId}
  - email, name, role (teacher|parent), classIds[], fcmToken

classes/{classId}
  - name, teacherId, teacherName, joinCode, memberIds[], description
  ├── messages/{messageId}
  │     - senderId, senderName, text, type (chat|announcement), timestamp
  ├── attendance/{studentId_date}
  │     - studentId, studentName, status (present|absent|late), date
  ├── grades/{gradeId}
  │     - studentId, studentName, subject, value, date, comment
  └── payments/{paymentId}
        - label, amount, dueDate, paymentStatuses: {userId: bool}
```

## 🔐 Firestore Security Rules (Recommended)

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can read/write their own profile
    match /users/{userId} {
      allow read, write: if request.auth.uid == userId;
    }

    // Any authenticated user can read classes they are a member of
    match /classes/{classId} {
      allow read: if request.auth.uid in resource.data.memberIds;
      allow create: if request.auth != null;
      allow update: if request.auth.uid in resource.data.memberIds;

      match /messages/{messageId} {
        allow read, create: if request.auth.uid in get(/databases/$(database)/documents/classes/$(classId)).data.memberIds;
        allow delete: if request.auth.uid == resource.data.senderId
          || request.auth.uid == get(/databases/$(database)/documents/classes/$(classId)).data.teacherId;
      }

      match /attendance/{attendanceId} {
        allow read, write: if request.auth.uid == get(/databases/$(database)/documents/classes/$(classId)).data.teacherId
          || request.auth.uid in get(/databases/$(database)/documents/classes/$(classId)).data.memberIds;
      }

      match /grades/{gradeId} {
        allow read, write: if request.auth.uid in get(/databases/$(database)/documents/classes/$(classId)).data.memberIds;
      }

      match /payments/{paymentId} {
        allow read, write: if request.auth.uid in get(/databases/$(database)/documents/classes/$(classId)).data.memberIds;
      }
    }
  }
}
```

## 📲 FCM Push Notifications

### Android
Add to `android/app/src/main/AndroidManifest.xml` inside `<application>`:
```xml
<meta-data
    android:name="com.google.firebase.messaging.default_notification_channel_id"
    android:value="high_importance_channel" />
```

### iOS
- Enable Push Notifications capability in Xcode
- Upload APNs key to Firebase Console → Project Settings → Cloud Messaging
